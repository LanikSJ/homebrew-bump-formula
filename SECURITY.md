# Security Policy

## Supported Versions

This GitHub Action is actively maintained and supports the latest version. Security updates will be provided for the
current version.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

We take security seriously in this project. If you discover a security vulnerability, please report it responsibly.

### How to Report

1. **Do not open a public issue** - This could expose the vulnerability before it's fixed
2. **Create a discussion or issue** on forum.lanik.us with details about the vulnerability
3. Include the following information:
   - Description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact
   - Any suggested fixes or mitigations

### Response Timeline

We are committed to addressing security reports promptly:

- **Acknowledgment**: Within 48 hours of receiving the report
- **Initial assessment**: Within 5 business days
- **Status update**: Every 7 days until resolution
- **Resolution**: As quickly as possible based on severity

### What to Expect

- We will acknowledge receipt of your report
- We will investigate and assess the vulnerability
- We will work on a fix in a timely manner
- We will coordinate with you on disclosure timing if appropriate
- We appreciate your patience during the resolution process

### Responsible Disclosure

We expect reporters to:

- Give us reasonable time to investigate and fix the issue before public disclosure
- Not exploit the vulnerability beyond what's necessary to demonstrate the issue
- Keep details confidential until we've had a chance to address it

Thank you for helping keep this project secure!
